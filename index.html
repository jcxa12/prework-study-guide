HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

HTML Elements

  • "p" = defines a paragraph on text
  • "a" = creates hyperlink to another web pages or resources
  • "ul" and "ol" = create unordered and ordered lists, respectively
  • "li" = represents list of items within "ul" and "ol" elements
  • "span" = a generic container for grouping and styling content

HTML Attributes

  • "href" (in "a") = specifies the URL or resource to link to
  • "src" (in "img") = defines the source URL for an image
  • "alt (in "img") = provides alternative text for images for accessibility
  • "id" = gives a unique identifier to an element, often used for CSS or JavaScript
  • "class" = assigns one or more classes to an element for styling purposes
  • "style" = allows inline CSS styling for individual elements
  • "target" (in "a") = specifies where the linked content will open
  • "type" (in "script" and "link") = defines the type of resources being linked or embedded

CSS

  • margin: indicates how much space we want around the outside of an element.
  • padding: indicates how much space we want around the content inside an element.
  • color: This property sets the color of the text.
  • font-family: You can use this property to choose the style of the font for your text, like "Arial," "Times New Roman," or "Verdana."
  • font-size: It helps you change the size of your text, making it bigger or smaller.
  • font-weight: You can make your text look bold (thicker) or normal with this property.
  • font-style: It lets you make your text italic or keep it normal (not slanted).
  • text-align: This one helps you decide if your text should align to the left, right, center, or be justified (even on both sides).
  • text-decoration: You can use it to add things like underlines or strikethrough lines to your text.
  • text-transform: This property changes the capitalization of your text, like making it all uppercase or lowercase.
  • line-height: You can set how much space there is between lines of text. It's like adjusting the space between ruled lines on a piece of paper.
  • letter-spacing: This one controls the space between letters. You can make your letters closer together or farther apart.
  • word-spacing: It changes the space between words, so you can make words closer or farther from each other.
  • text-shadow: You can add a shadow behind your text to make it stand out.

Git

  • git init - "Start a Magic Notebook": It's like starting a special notebook for your drawings.
  • git clone - "Copy Friend's Notebook": You can make a copy of your friend's notebook with all their drawings.
  • git add - "Pick and Put on Shelf": Choose a drawing and put it on a special shelf so you don't forget it.
  • git commit -m "message" - "Save with Note": When you're done drawing, you save your work and leave a little note to remember what you did.
  • git status - "Check the Shelf": You look at the special shelf to see what's there and what you still need to save.
  • git pull - "Get New Drawings": If your friend has new drawings, you can ask them to share with you.
  • git push - "Give Your Drawings": If you have new drawings, you can share them with your friend.
  • git branch - "Different Magic Notebooks": You can have different notebooks for different kinds of drawings.
  • git checkout - "Change Notebook": You can switch to a different notebook for different drawings.
  • git merge - "Mix Drawings": If you have drawings in two notebooks, you can mix them together.
  • git log - "History Book": It's like a book that tells you a story about all the drawings you made.
  • git stash - "Hide Drawings": Sometimes you want to hide your drawings for a while and then bring them back.
  • git remote -v - "Friends' Notebooks": You can see if your friends have notebooks like yours.
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • Variables: These are like labeled jars where you can keep things you want to remember.
  • Data Types: It's like knowing if something is a number, a word, or a yes/no answer.
  • Operators: These are signs and words like + (plus), - (minus), and ? (question mark) to do math and ask questions.
  • Conditional Statements: It's like making choices, such as deciding whether to go outside or stay inside.
  • Loops: These are like doing something over and over, such as playing your favorite game again and again.
  • Functions: Think of them as special recipes that help you do specific tasks.
  • Arrays: They're like lists where you put many things together, just like your collection of toys.
  • Objects: Objects are like toys, each with a name and special features.
  • Events: These are like when you push buttons or play with toys to make something happen.
  • DOM Manipulation: It's about changing how things look on a website, like moving your toys around in your room.
  • Error Handling: Think of it like getting help when you make a mistake or can't find your toys.
  • Scope: It's about where you can play with your toys; some toys are only for your room, and some are for the whole house.
  • Closures: Closures are like secrets, where you keep things hidden and safe.
  • Asynchronous Programming: This is when you can do other things while you wait for something to finish, like a grown-up doing chores while you play.
  • AJAX and Fetch: It's like asking someone to get you a cookie from the kitchen without leaving your room.
  • Modules: Modules are like separate boxes for different toys, so you can find them easily.
  • JSON: It's a way to write down information, like a story about your favorite toy using words and pictures.
